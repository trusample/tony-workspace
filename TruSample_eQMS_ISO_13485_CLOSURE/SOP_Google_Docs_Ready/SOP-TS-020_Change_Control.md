SOP-TS-020 — Change Control Procedure
Purpose
Establish a controlled process for submitting, assessing, implementing, and closing changes that affect product, processes, documents, software automations, or the eQMS.

Scope
All changes that may affect product quality, regulatory compliance, or the integrity of records.

Definitions
- Change Request (CR): formal request recorded in FRM-TS-203
- Mgmt Action: assigned task(s) for implementing or verifying change

Procedure
1. Submit Change Request (FRM-TS-203)
   - Capture: Change ID, Requester, Date, Description, Risk Level, Owner, Affected Items.
2. Triage & Risk Assessment
   - Quality Reviewer assesses risk; determine need for validation or CAPA.
3. Plan & Approve
   - Prepare implementation plan, assign owners and due dates; obtain approval.
4. Implement
   - Execute changes (code deploys, doc edits, process updates). Record evidence (diffs, screenshots).
5. Verify & Validate
   - Perform required validation per plan; record results in validation artifact and link to CHANGE ID.
6. Close Change
   - Update FRM-TS-203: status=Closed; record approver and date; update Documents_Index if controlled docs changed.

Records
- FRM-TS-203 Change Log entries
- FRM-TS-204 Mgmt Action records
- Validation artifacts (VAL-ENG-*)
